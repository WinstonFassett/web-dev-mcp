# click() doesn't trigger Next.js Link navigation — need MouseEvent dispatch

## Problem

`browser.click()` calls `el.click()` on the found element. For Next.js `<Link>` components (rendered as `<a>` tags), this doesn't trigger client-side navigation. The URL doesn't change and the page stays on the current view.

This was discovered clicking Monitor sub-tabs (Visibility, Prompts, Recommendations, Performance) and the Activate sidebar link in CommerceOS.

## Root cause

Next.js Link components use React's synthetic event system. The native `el.click()` method fires a click event but it doesn't have the properties React's event delegation expects (e.g., `view`, `bubbles` in the right shape). The Link component's click handler checks these properties to decide whether to use client-side navigation vs full page load.

## What works

Dispatching a proper `MouseEvent` with `bubbles: true, cancelable: true, view: window`:

```js
el.dispatchEvent(new MouseEvent('click', { bubbles: true, cancelable: true, view: window }))
```

This triggers Next.js client-side navigation correctly.

## Fix

In `packages/gateway/src/client/index.ts`, the `click()` method (line ~461) should use `dispatchEvent(new MouseEvent(...))` instead of `el.click()`:

```js
click(selector: string) {
  const el = this.findElement(selector)
  if (!el) return { error: 'Element not found: ' + selector }
  el.dispatchEvent(new MouseEvent('click', { bubbles: true, cancelable: true, view: window }))
  return { clicked: selector, tag: el.tagName.toLowerCase() }
}
```

Also applies to any other interaction methods that use `el.click()`.

## Impact

Any SPA with client-side routing (Next.js, React Router, Vue Router, SvelteKit) may have this issue. The `el.click()` fallback works for buttons and form elements but not for framework navigation links.
